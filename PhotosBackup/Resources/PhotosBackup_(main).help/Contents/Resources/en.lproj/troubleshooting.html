Troubleshooting

Common Errors and Possible Solutions

Please grant photos access macOS 10.14 requires the user to explicitly approve applications to access the user's Photos Library in order to protect the user's privacy. You can resolve this issue by clicking on Open System Preferences in the alert prompt, which will take you to the Security->Privacy pane where you can approve or deny permission.
Invalid photos library The selected file isn't a macOS Photos library. At this time Photos Backup only backs up macOS photo library packages which end in .photoslibrary.
No photos library could be found at the specified location A photos library you previously selected and which the app could access is no longer accessible. If it was on a removable or network drive, that drive might have been disconnected or unmounted, in which case try and reconnect to the drive. Otherwise, if stored internally, the path may have changed, or the library may have been deleted.
Unable to create backup target Photos Backup was unable to automatically create a sparsebundle in the selected directory. This could be because there is not enough space on the selected volume, or you don't have write permission. Try selecting another location, or ensuring there is enough space to create a new file on the selected drive, and try again.
Invalid sparsebundle The selected backup target is not a sparsebundle format disk image. Photos Backup needs a valid sparsebundle in order to mount it as a volume and run backups. Please make sure you've selected a valid sparsebundle, or create a new one using 'Create new backup location…'
The backup location could not be found Check to see whether a sparsebundle exists at the backup location you specified, and whether it has mounted correctly. If it has mounted correctly you should be able to visit the path /Volumes/photosBackup' in the Finder. To visit a custom path, open the Finder and press ⌘⇧G to open the Go to Folder… dialog. If the sparsebundle` is located on a network drive, check that the network drive is accessible. Photos Backup will try and remount the network drive if it has been disconnected, but may not be able to succeed if e.g., you are no longer on the same network.
The backup location is too small The maximum size of the sparsebundle is set to one that is smaller than the photos library you are trying to back up. Check that the drive on which the sparsebundle file is located is large enough to accommodate your photos library. Then, you can either use the hdiutil utility in macOS to resize the sparsebundle manually, or create a new sparsebundle using the app. Photos Backup will include the facility to enlarge sparsebundles automatically in a future release
This application is already running You have attempted to launch a second instance of Photos Backup. If you want to restart the app, first quit the existing instance, either by using the menu bar or killing the app in Activity Monitor or the command line. The error will show you the PIDs of all running Photos Backup processes to help you with this.

Advanced Troubleshooting

If the issue isn't solved by common errors and solutions, you may find more information about the error in the detailed log, found under the Log pane. From here you can read debug messages as well as the full output of the rsync process and attempt to resolve errors, or export the log.
